AI Inventory Demand Forecasting System

A machine learning web application built with XGBoost and Flask to predict daily inventory demand from store, item, and date inputs.

Model XGBoost regressor trained on retail demand data with engineered time and aggregate features.
Purpose Support SME-style inventory planning with lightweight AI-based forecasting.
Tech Stack Python, Pandas, Scikit-learn, XGBoost, Flask, HTML, CSS, JavaScript.

About This Project

This application is based on my final-year dissertation project focused on improving inventory decision-making for SMEs using machine learning. The system uses engineered historical sales features and an XGBoost model to generate demand forecasts through a lightweight Flask interface.

Note Predictions are based on historical demand patterns and should support business decisions, not replace them.

API Example

The application also exposes a simple JSON prediction endpoint.

POST /predict

Send a JSON payload with store, item, and date.

{
  "store": 1,
  "item": 1,
  "date": "2017-04-10"
}

Example Response

{
  "store": 1,
  "item": 1,
  "date": "2017-04-10",
  "predicted_sales": 13.42
}

You can test the API route locally after running the Flask app.
